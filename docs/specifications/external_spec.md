# 定数管理システム 外部仕様書

**版番:** 1.0  
**作成日:** 2026年1月21日  
**対象システム:** 車載ソフトウェア定数管理ツール

---

## 1. システム概要

### 1.1 目的
本システムは、車載ソフトウェアの定数をGUIで一元管理し、CSVファイルを用いた定数データの入出力を可能にするデスクトップアプリケーションです。開発メンバー間での定数値の共有、外部編集、バージョン管理を効率化します。

### 1.2 システム構成
- **GUI 管理画面**: Windowsフォームベースの定数管理インターフェース
- **CSV入出力機能**: 定数データのエクスポート・インポート
- **ファイル保存**: 定数データの永続化
- **エラーハンドリング**: 不正なデータ形式や型不一致への対応

### 1.3 主なユーザー
- 車載ソフトウェア開発エンジニア
- システムテスト担当者
- プロジェクトマネージャー

---

## 2. データ構造

### 2.1 定数アイテム (Constant Item)

定数管理システムで扱う最小単位のデータ構造を以下に定義します：

| 項目名 | 型 | 必須 | 制限・備考 |
|--------|-----|------|----------|
| **ID** | 文字列 | ◎ | システム内で一意。英数字とアンダースコアのみ。最大32文字。キー項目。 |
| **Name** | 文字列 | ◎ | 定数の日本語名称。最大64文字。 |
| **Value** | 数値/文字列 | ◎ | 設定値。数値またはテキスト。 |
| **Unit** | 文字列 | ○ | 単位情報（例："℃", "km/h", "V"）。最大16文字。省略可能。 |
| **Description** | 文字列 | ○ | 定数の説明・用途。最大256文字。省略可能。 |

**注記:**
- ID は CSVインポート時の主キーとして機能
- Value は数値、小数、負数、または文字列を受け付ける
- 日本語を含むテキストは UTF-8 で処理

### 3.2 CSVエクスポート機能

#### 3.2.1 仕様

- **対象データ**: ユーザーがチェックボックスで選択した行のみをCSV形式で出力
- **ヘッダー行**: 必ず1行目に各カラム名を含める
- **文字コード**: UTF-8（BOM付き必須）
- **改行コード**: CRLF（Windows標準）
- **ファイル形式**: `.csv`
- **デリミタ**: カンマ `,`

#### 3.2.2 出力例

```csv
ID,Name,Value,Unit,Description
ENGINE_TEMP_MAX,エンジン最高温度,120,℃,エンジンの最大許容温度
SPEED_LIMIT,速度制限,100,km/h,法定速度制限
BATTERY_VOLTAGE,バッテリー電圧,12.0,V,正常動作時の基準電圧
```

#### 3.2.3 特殊文字の処理

- CSVデータ内にカンマやダブルクォートを含む場合は、ダブルクォートで囲む
- 改行を含む場合はダブルクォートで囲む

**例:**
```csv
ID,Name,Value,Unit,Description
SENSOR_RANGE,"センサー検出範囲","0-100","%","正常範囲: 0～100"
```

### 3.3 CSVインポート機能

#### 3.3.1 仕様

- **ファイル形式**: `.csv` ファイル
- **文字コード**: UTF-8（BOM付き必須）、SHIFT_JIS にも対応
- **処理ロジック**:
  1. CSVファイルを読み込む
  2. ヘッダー行をスキップ
  3. 各データ行に対し:
     - **ID が既に存在** → 既存行を更新
     - **ID が新規** → 新規行として追加
- **マージモード**: 既存データと新規データをマージ（デフォルト動作）
- **置換モード**: 既存データをすべて削除して新データで置換

#### 3.3.2 インポート時の検証

| 検証項目 | 内容 | エラー時の動作 |
|---------|------|----------------|
| **ヘッダー行** | ID, Name, Value, Unit, Description が存在することを確認 | エラー表示、インポート中止 |
| **必須項目** | ID, Name, Value が空でないことを確認 | その行をスキップ、警告表示 |
| **ID 形式** | 英数字とアンダースコアのみ、最大32文字 | エラー表示、その行をスキップ |
| **Value 型** | 数値または文字列を判定（自動判別） | そのまま受け入れ |
| **重複チェック** | 同一ファイル内での ID 重複を検出 | エラー表示、後の行を優先 |

### 3.4 エラー処理

#### 3.4.1 エラーメッセージ一覧

| エラーコード | メッセージ | 原因 | ユーザーアクション |
|------------|-----------|------|------------------|
| **E001** | "ファイルが見つかりません" | 指定ファイルが存在しない | ファイルパスを確認 |
| **E002** | "ファイルの読み込みに失敗しました" | ファイルロック、権限不足 | ファイルを閉じるか権限を確認 |
| **E003** | "不正なCSV形式です（ヘッダー行が不正）" | ヘッダー行が仕様と異なる | CSVファイル形式を確認 |
| **E004** | "エンコーディングが認識できません" | サポート外の文字コード | UTF-8 または SHIFT_JIS に変換 |
| **E005** | "データ行 {行番号}: ID が空です" | 必須項目が欠落 | その行の ID を入力 |
| **E006** | "データ行 {行番号}: Name が空です" | 必須項目が欠落 | その行の Name を入力 |
| **E007** | "データ行 {行番号}: Value が空です" | 必須項目が欠落 | その行の Value を入力 |
| **E008** | "データ行 {行番号}: ID の形式が不正です（英数字とアンダースコアのみ）" | ID が仕様を逸脱 | 正しい形式に修正 |
| **E009** | "データ行 {行番号}: ID の長さが上限を超えています（最大32文字）" | ID が長すぎる | 32文字以内に短縮 |
| **E010** | "データ行 {行番号}: 重複する ID があります（{ID}）" | 同一ファイル内で ID が重複 | ID を修正 |
| **E011** | "保存に失敗しました（ディスク容量不足）" | ストレージ容量不足 | ストレージを確認・清掃 |
| **E012** | "ファイルへの書き込み権限がありません" | 権限不足 | ファイルの権限設定を確認 |
| **W001** | "警告: データ行 {行番号} は必須項目が欠落しているためスキップされました" | 不完全な行がある | 該当行を確認・修正 |
| **W002** | "警告: {n} 行のデータがインポートされました。{m} 行はスキップされました" | 一部の行がスキップされた | 結果を確認 |

#### 3.4.2 エラーダイアログ例

```
[エラー]
───────────────────────────────
データ行 5: ID の形式が不正です
（英数字とアンダースコアのみ）

ID: "Engine-Temp" は無効な文字を含みます

[OK]
───────────────────────────────
```

## 5. ユースケースフロー

### 5.1 UC-001: 定数を修正してCSV出力する

**前提条件**: 定数管理システムが起動している

**主シーン**:

| ステップ | アクター | アクション | システム応答 |
|---------|---------|-----------|-----------|
| 1 | ユーザー | [読込]ボタンをクリック | ファイル選択ダイアログが表示 |
| 2 | ユーザー | 既存のCSVファイルを選択し、[開く] | CSV ファイルを読み込みして検証 |

## 6. ファイル仕様

### 6.1 CSVファイル形式仕様

#### 6.1.1 基本情報
- **拡張子**: `.csv`
- **文字コード**: UTF-8（BOM付き必須）、SHIFT_JIS にも対応
- **改行コード**: CRLF（Windows標準）
- **デリミタ**: カンマ `,`
- **テキスト囲み文字**: ダブルクォート `"`

#### 6.1.2 ファイル構造

```
行番号 | 内容
────────────────────────────────────────────────────
1      | ID,Name,Value,Unit,Description
2      | CONST_001,定数名1,値1,単位1,説明1
3      | CONST_002,定数名2,値2,単位2,説明2
...    | ...
n      | CONST_nnn,定数名n,値n,単位n,説明n
```

#### 6.1.3 カラム仕様表

| カラム番号 | カラム名 | データ型 | 最大長 | 必須 | 備考 |
|-----------|---------|---------|--------|------|------|
| 1 | ID | String | 32 | ◎ | 英数字・アンダースコアのみ |
| 2 | Name | String | 64 | ◎ | 日本語対応 |
| 3 | Value | String | 256 | ◎ | 数値、小数、文字列対応 |
| 4 | Unit | String | 16 | ○ | 単位表示（省略可） |
| 5 | Description | String | 256 | ○ | 説明文（省略可） |

#### 6.1.4 データ型と表現例

**Value カラムの例**:
- 整数: `100`, `-50`, `0`
- 小数: `3.14`, `-2.5`, `0.001`
- 文字列: `"HIGH"`, `"ON"`, `"abc123"`
- 特殊: `""` （空文字列）

**Unit カラムの例**:
- 温度: `℃`, `°C`, `K`
- 速度: `km/h`, `m/s`, `mph`
- 電圧: `V`, `mV`
- パーセント: `%`
- その他: `rpm`, `Hz`, `kPa`

#### 6.1.5 テキスト囲み処理の例

**カンマを含む場合**:
```csv
ID,Name,Value,Unit,Description
RANGE_DEF,"検出範囲","0,100","%",説明
```

**改行を含む場合**:
```csv
ID,Name,Value,Unit,Description
SENSOR_NOTE,"センサー説明","値","単位","1行目
2行目"
```

**ダブルクォートを含む場合**:
```csv
ID,Name,Value,Unit,Description
FORMAT_STR,"フォーマット","\"VALUE\": %d","unit",説明
```

```csv
SYSTEM_MODE,システムモード,AUTO,,自動/手動選択モード
```

##### 6.2.2 選択行のみ出力例

ユーザーが ID, Name, Value の3行のみを選択した場合：

```csv
ID,Name,Value,Unit,Description
ENGINE_TEMP_MAX,エンジン最高温度,120,℃,エンジンの最大許容温度
SPEED_LIMIT,速度制限,100,km/h,法定速度制限値
BATTERY_VOLTAGE_NOMINAL,バッテリー基準電圧,12.0,V,正常動作時の基準電圧値
```

## 7. エラーメッセージ一覧

### 7.1 エラーメッセージ詳細定義

#### ファイルI/O エラー

| コード | メッセージ | 説明 | 対処方法 |
|--------|-----------|------|---------|
| **E001** | ファイルが見つかりません | 指定したファイルが存在しない | ファイルパスを確認してください |
| **E002** | ファイルの読み込みに失敗しました | ファイルがロック状態またはアクセス権限なし | ファイルが開いていないか確認し、アクセス権限を確認 |
| **E011** | 保存に失敗しました（ディスク容量不足） | ストレージ容量が不足 | ディスク容量を確認・解放してください |
| **E012** | ファイルへの書き込み権限がありません | ファイルまたはディレクトリへの書き込み権限がない | ファイルの権限設定を確認してください |

#### CSV形式エラー

| コード | メッセージ | 説明 | 対処方法 |
|--------|-----------|------|---------|
| **E003** | 不正なCSV形式です（ヘッダー行が不正） | ヘッダー行が仕様と異なる | CSVファイルのヘッダー行が「ID,Name,Value,Unit,Description」か確認 |
| **E004** | エンコーディングが認識できません | サポート外の文字コード | ファイルを UTF-8 または SHIFT_JIS に変換 |

#### データバリデーション エラー

| コード | メッセージ | 説明 | 対処方法 |
|--------|-----------|------|---------|
| **E005** | データ行 {行番号}: ID が空です | ID フィールドが空欄 | {行番号} 行の ID を入力してください |
| **E006** | データ行 {行番号}: Name が空です | Name フィールドが空欄 | {行番号} 行の Name を入力してください |
| **E007** | データ行 {行番号}: Value が空です | Value フィールドが空欄 | {行番号} 行の Value を入力してください |
| **E008** | データ行 {行番号}: ID の形式が不正です（英数字とアンダースコアのみ） | ID に不正な文字が含まれている | ID は英数字（a-z, A-Z, 0-9）とアンダースコア（_）のみ使用可能 |
| **E009** | データ行 {行番号}: ID の長さが上限を超えています（最大32文字） | ID が32文字を超えている | ID を32文字以内に短縮してください |
| **E010** | データ行 {行番号}: 重複する ID があります（{ID}） | 同一ファイル内で同じ ID が存在 | {ID} を一意な ID に変更してください |

## 8. 非機能要件

### 8.1 パフォーマンス

- CSV インポート（1000行）: 1秒以内に完了
- グリッド表示: スクロール時のレスポンス 100ms以内
- エクスポート: 100行程度で 500ms以内

### 8.2 セキュリティ

- ファイルアクセスは Windowsのファイルシステムパーミッションに準ずる
- ユーザー入力値は XSS対策として無効化処理を施す
- パスワードなどの機密情報はValue に含めない（ユーザー責任）

### 8.3 ユーザビリティ

- 日本語UI対応
- キーボードショートカット対応（例：Ctrl+S で保存）
- アンドゥ・リドゥ機能（複数ステップ対応）

### 8.4 保守性

- エラーログは `%APPDATA%\ConstantManager\logs\` に保存
- ログファイル: `YYYYMMDD.log`
- ログレベル: INFO, WARN, ERROR
